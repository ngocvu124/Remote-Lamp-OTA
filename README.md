# 🌟 Smart Remote Lamp Controller (ESP32-S3 + LVGL)

![Platform](https://img.shields.io/badge/platform-ESP32--S3-orange.svg)
![Framework](https://img.shields.io/badge/framework-Arduino-00979C.svg)
![UI](https://img.shields.io/badge/UI-LVGL_8-brightgreen.svg)

A highly optimized, feature-rich smart remote controller built with **ESP32-S3**, featuring a beautiful **LVGL graphical interface**, a shared-SPI SD Card system, and an embedded Web File Manager. It is designed to control a custom LED lamp via **ESP-NOW** protocol with ultra-low latency.

## ✨ Key Features

* 🎛️ **Intuitive UI:** Custom graphical interface designed with EEZ Studio, powered by LVGL 8. Navigate smoothly using a Rotary Encoder.
* 📡 **ESP-NOW Control:** Ultra-fast, routerless communication to control lamp brightness (0-100%) and color temperature (2700K-6500K).
* 🖼️ **Dynamic Backgrounds:** Load custom RGB565 `.bin` image backgrounds directly from the MicroSD card.
* 🌐 **Embedded Web Server:** A fully functional, responsive Web File Manager to upload, download, and recursively delete files/folders on the SD card via WiFi.
* 📈 **Real-time Stock Monitor:** Track customized stock/crypto tickers directly on the remote screen.
* ☁️ **OTA Updates:** Hassle-free Over-The-Air firmware updates directly from GitHub raw binaries.
* 🔋 **Power Management:** Deep sleep mode with adjustable timeout, OLED backlight control, and real-time battery voltage/capacity monitoring.
* 🛡️ **Rock-Solid Hardware Handling:** Advanced handling of Shared SPI bus between the TFT display and SD Card, featuring auto-recovery (`wakeupSD`) and SRAM buffering to prevent DMA PSRAM tearing.

---

## 🛠️ Hardware Requirements

* **Microcontroller:** ESP32-S3 (e.g., Wemos Lolin S3 Mini)
* **Display:** 1.54" or 1.3" ST7789 IPS TFT Display (240x240)
* **Storage:** MicroSD Card Module
* **Input:** Rotary Encoder (with push-button)
* **Power:** 3.7V LiPo Battery with a voltage divider circuit for monitoring.

## 📁 SD Card Structure

The MicroSD card must be formatted to **FAT32**. 
For the UI to load backgrounds correctly, create a folder named `/background` in the root of your SD card and place your RGB565 `.bin` image files (240x240 pixels, ~115,200 bytes) inside.

```text
SD_ROOT/
│
├── background/         # Put your custom background .bin files here
│   ├── bg1.bin
│   └── bg2.bin
│
└── config.json         # Auto-generated by the system
